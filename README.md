# PCR_bias

Open-source workflow draft for analyzing PCR amplification bias in mammalian mitochondrial 12S/16S amplicon sequencing.

This repository contains thesis analysis scripts for two related workflows:

- **Chapter 2–3 workflow**: sequencing-data preprocessing, abundance table construction, sequence annotation, species-label harmonization, threshold sensitivity analysis, sequence-feature extraction, and univariate statistics.
- **Chapter 4 workflow**: PCR/non-PCR comparative analysis, sequence and k-mer feature extraction, paired statistical tests, machine-learning dataset preparation, Random Forest/XGBoost/1D-CNN modeling, SHAP/integrated-gradient interpretation, external validation, and paper-ready table export.

The current repository is a **public-release and thesis-provenance draft**. Legacy scripts are preserved under `legacy/` so the thesis analyses remain traceable. New files under `src/`, `tools/`, `tests/`, and `docs/` provide a safer public-facing layer for schema checking, metadata consistency, synthetic smoke testing, and future refactoring.

## Installation

For the full analysis environment:

```bash
conda env create -f environment.yml
conda activate pcr-bias-pipeline
```

For lightweight repository checks only:

```bash
python -m pip install --upgrade pip pytest
```

The lightweight checks do not require the full scientific stack because they avoid importing most legacy analysis dependencies.

## Sanity checks

Run all public lightweight checks:

```bash
make ci
```

Equivalent commands:

```bash
bash tools/smoke_test.sh
python tools/check_repository_consistency.py
python tools/audit_hardcoded_paths.py --legacy-mode report
python tools/run_synthetic_smoke.py
python -m pytest
```

`tools/smoke_test.sh` checks Python and Shell/Slurm syntax. `tools/run_synthetic_smoke.py` creates a temporary artificial Chapter 4 project and verifies that the manifest builder and single-end preprocessing scripts can run on tiny FASTQ inputs. These checks validate parser/schema/path wiring only; they do not validate scientific conclusions.

## Recommended entry points

### Chapter 2–3

```bash
cd legacy/chapter2_3
bash 27_run_chapter3_pipeline_resume.sh /path/to/chapter2_3_analysis 48 /path/to/RNAfold
```


### Chapter 4

Chapter 4 remains a legacy collection rather than a single polished CLI. Treat the `00*`–`08*` canonical/support scripts as the recommended internal order and keep `ext*` scripts as an external-validation branch.

Detailed order and expected outputs are documented in:

```text
docs/RUN_ORDER.chapter4.tsv
```

## Input schemas

The two workflows use different metadata schemas. See:

```text
docs/SCHEMAS.md
```

Key Chapter 4 sample metadata columns are:

```text
file_id, sample_name, species, n_individuals, locus, pcr
```

Key Chapter 2–3 sample metadata columns are:

```text
sample_id, marker, group_name, sample_type, species_scope, is_core_analysis
```

## Configuration

Copy the example configuration and edit local paths:

```bash
cp configs/config.example.yaml configs/config.yaml
```

Do not commit `configs/config.yaml`, private metadata, raw sequencing data, BAM files, trained models, logs, or large intermediate tables.

## Data availability and excluded files

This repository intentionally excludes original large sequencing data and most generated intermediates. See:

```text
docs/DATA_AVAILABILITY.md
docs/OUTPUTS.md
```

Synthetic records under `examples/synthetic/` and the temporary data generated by `tools/run_synthetic_smoke.py` are artificial and are only for parser/path-wiring checks.

## Development notes

Before committing changes, run:

```bash
make ci
```

When adding or removing Chapter 4 scripts, update `docs/CHAPTER4_SCRIPT_STATUS.tsv`; the consistency checker flags missing or stale status entries. When changing public metadata, keep the repository URL fixed as:

```text
https://github.com/SDUMCWenchao/PCRbias
```

## Public-release status

- Legacy private server paths are expected to be replaced by public placeholders or local configuration values.
- Chapter 4 script status is documented in `docs/CHAPTER4_SCRIPT_STATUS.tsv`.
- Tiny synthetic checks are included for parser and path-wiring validation.
- Dependency baselines are pinned in `requirements.txt` and mirrored in `environment.yml` where practical.
- Legacy scripts remain available for thesis provenance; future work should promote shared logic into `src/pcr_bias/` and add stable CLI entry points.
